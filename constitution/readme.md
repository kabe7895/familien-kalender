# 2. Projektbeschreibung

FamilyFlow ist ein digitaler Familienkalender zur Organisation von:

- Terminen
- Aufgaben
- Erinnerungen
- Verfügbarkeiten von Familienmitgliedern

Ziel ist eine einfache, intuitive und datenschutzfreundliche Plattform für Familien zur besseren Alltagskoordination.

---


# 3. Requirements Engineering

Für das Projekt wurde Requirements Engineering mit zwei Tools durchgeführt:

---

## 3.1 Self-Made Tool (Notion Simulation)

Requirements wurden in einer tabellarischen Notion-Datenbank modelliert.

Enthaltene Attribute:

- ID
- Titel
- Beschreibung
- Typ (funktional / nicht-funktional)
- Priorität (Must / Should / Could)
- Status
- Stakeholder (Quelle: kb)
- Akzeptanzkriterien

Beispiel-Requirements:

- R4 – Verfügbarkeitsanzeige (kb)
- R7 – Datenschutz & Datenminimierung (kb)

---

## 3.2 Professionelles Tool (GitHub Issues Simulation)

Requirements wurden zusätzlich als GitHub Issues modelliert.

Jedes Issue enthält:

- Titel mit Quelle (kb)
- Beschreibung
- Akzeptanzkriterien
- Labels (functional / non-functional / priority)
- Abhängigkeiten (falls vorhanden)

Beispiel-Requirements:

- R4 (kb): Verfügbarkeitsanzeige
- R7 (kb): Datenschutz

---

# 4. Vision des Projekts

FamilyFlow soll langfristig eine intelligente Familienplattform werden mit:

- automatischer Terminoptimierung
- Konflikterkennung
- KI-gestützter Tagesplanung
- Lernfähigkeit (Zeitbedarf von Familienmitgliedern)
- transparenter und erklärbarer Automatisierung

---

# 5. Nicht-Ziele

- kein soziales Netzwerk
- keine Werbeplattform
- keine Business-Software
- keine überladene App
- kein Tracking für Werbung

---

# 6. Fazit

Das Projekt demonstriert:

- Requirements Engineering mit zwei Werkzeugen
- strukturierte Vision, Roadmap und Tech Stack
